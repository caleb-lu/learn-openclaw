# Lesson 9: Prompt Engineering

OpenClaw assembles every LLM request from an eight-layer prompt stack. Understanding this stack gives you precise control over what the model sees and in what order. This lesson breaks down each layer, explains precedence rules, and demonstrates effective prompt design patterns.

## Learning Objectives

- Understand the 8-layer prompt stack and how it is assembled
- Learn the layer order and precedence rules
- Distinguish between system prompts and user prompts
- Use workspace files as prompt layers
- Apply effective prompt design patterns

## Prerequisites

- Completion of Lesson 8 (Multi-Agent Setup)
- Understanding of workspace files (Lesson 3)

## The 8-Layer Prompt Stack

Every time the LLM is called, OpenClaw constructs a prompt by stacking eight layers in a fixed order. Layers are concatenated from bottom to top, with later layers appearing closer to the user's actual message:

| Layer | Number | Source              | Description                        |
|-------|--------|---------------------|------------------------------------|
| 8     | Bottom | Global System       | Framework-level instructions       |
| 7     |        | SOUL.md             | Core personality                   |
| 6     |        | IDENTITY.md         | Name, tone, expertise              |
| 5     |        | TOOLS.md            | Tool descriptions and rules        |
| 4     |        | SKILL.md files      | Per-skill instructions             |
| 3     |        | MEMORY.md           | Recalled memory entries            |
| 2     |        | Session History     | Prior messages in the conversation |
| 1     | Top    | User Message        | The current input                  |

Layer 1 (top) is closest to the model's generation point, so it has the strongest influence on the immediate response. Layer 8 (bottom) sets the foundational context. The model sees the entire stack as a single conversation, with system layers playing the role of persistent instructions.

## Layer Order and Precedence

Layers are not simply concatenated. OpenClaw uses structured message roles to communicate precedence to the LLM:

```
[System] Layer 8: Global system instructions
[System] Layer 7: SOUL.md content
[System] Layer 6: IDENTITY.md content
[System] Layer 5: TOOLS.md content
[System] Layer 4: SKILL.md content (one message per skill)
[System] Layer 3: MEMORY.md relevant entries
[User]   Previous user messages...
[Assistant] Previous assistant messages...
[User]   Layer 1: Current user message
```

System messages carry high precedence in most LLMs. When a system message says "do not use emojis" and a user message says "use lots of emojis," the model typically follows the system instruction. This is why personality definitions in SOUL.md reliably override casual user requests.

You can preview the exact prompt stack for your configuration with:

```bash
openclaw prompt preview --verbose
```

This prints every layer with its token count and source file, so you can identify which layers consume the most context.

## System Prompt vs User Prompt

OpenClaw separates prompts into two categories:

- **System prompts** (layers 3-8) are injected as `role: "system"` messages. They define behavior, constraints, and context.
- **User prompts** (layers 1-2) are injected as `role: "user"` messages. They represent actual conversation turns.

You should never put user content into system layers or vice versa. Mixing them confuses the model about what is an instruction and what is a conversational input.

### The Global System Layer (Layer 8)

Layer 8 is generated by the framework itself. You do not edit it directly, but you can customize parts of it through configuration:

```json5
{
  prompt: {
    // Prepend custom instructions before all workspace files
    preamble: "You must respond in English unless the user writes in another language.",
    // Append instructions after all workspace files but before memory
    postamble: "Always end your response with a brief summary.",
    // Maximum tokens allocated to the prompt stack
    maxSystemTokens: 2048,
  },
}
```

## Workspace Files as Prompt Layers

Each workspace Markdown file maps directly to a prompt layer. The filename determines which layer it occupies:

- `SOUL.md` -> Layer 7
- `IDENTITY.md` -> Layer 6
- `TOOLS.md` -> Layer 5
- Custom `.md` files in `workspace/layers/` -> Layer 4 (alphabetical order)

You can add arbitrary layers by placing Markdown files in the `workspace/layers/` directory:

```bash
# Add a custom knowledge layer
mkdir -p workspace/layers
cat > workspace/layers/PRODUCT_KB.md << 'EOF'
# Product Knowledge Base

Our product uses a microservices architecture with the following services:
- auth-service: JWT-based authentication, port 8001
- api-gateway: REST API routing, port 8000
- worker: Background job processing, port 8002
EOF
```

Custom layers are loaded between TOOLS.md and MEMORY.md, giving them moderate precedence. They are ideal for injecting domain knowledge that the assistant should reference but that does not belong in the personality definition.

## Effective Prompt Design Patterns

Here are patterns that consistently produce good results with OpenClaw's stack:

### Pattern 1: Negative Instructions in SOUL.md

```markdown
## Constraints

- Do not hallucinate URLs or version numbers.
- Do not claim to have real-time data unless a tool provides it.
- Do not begin responses with "Sure!" or "Of course!"
```

### Pattern 2: Structured Tool Descriptions in TOOLS.md

```markdown
## web_search

**When to use:** The user asks about current events, prices, or
any fact that may have changed after your training cutoff.

**When NOT to use:** General knowledge questions, creative writing,
or when the answer is in your training data.

**Output format:** Always cite the source URL.
```

### Pattern 3: Few-Shot Examples in Custom Layers

```markdown
# Response Examples

## Example 1
User: "How do I reset my password?"
Assistant: "Go to Settings > Account > Security and click 'Reset
Password'. You will receive an email with a reset link valid for
24 hours."

## Example 2
User: "The app is crashing."
Assistant: "I'm sorry to hear that. Could you tell me:
1. Which version of the app are you using?
2. What device and OS?
3. Does it crash on a specific action?
This will help me narrow down the issue."
```

### Pattern 4: Token Budget Awareness

Keep workspace files concise. Every token in the system prompt reduces the space available for conversation history. Use `openclaw prompt preview` to monitor token usage:

```
Layer 8 (System):     120 tokens
Layer 7 (SOUL.md):    340 tokens
Layer 6 (IDENTITY.md): 210 tokens
Layer 5 (TOOLS.md):   480 tokens
Layer 4 (Skills):     290 tokens
Layer 3 (Memory):     150 tokens
Total system:         1,590 tokens
Remaining for conversation: 2,506 tokens (4096 context)
```

If your system layers consume too many tokens, consider trimming workspace files or moving lower-priority content into MEMORY.md (which is selectively injected rather than always included).

## Summary

The eight-layer prompt stack gives you fine-grained control over how the LLM perceives context. Personality goes in SOUL.md, expertise in IDENTITY.md, tool rules in TOOLS.md, and domain knowledge in custom layer files. Understanding this stack is the key to crafting an assistant that behaves consistently and reliably. In the next lesson we will explore the memory system, which allows your assistant to learn and remember across sessions.
